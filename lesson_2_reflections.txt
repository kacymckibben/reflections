What happens when you initialize a repository? Why do you need to do it?

It creates the repository. If you don’t run it, it’s just a regular folder. 

How is the staging area different from the working directory and the repository? What value do you think it offers?

The staging area acts as the go-between between the working directory and the repository. This allows you to better organize your commits.

How can you use the staging area to make sure you have one commit per logical change?

You can use the staging area to organize your changes so that if you have made multiple changes, you can add the ones that make sense to add in one commit, then commit that. then repeat.
